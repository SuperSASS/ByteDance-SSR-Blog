# 作业要求 - SSR 博客系统

## 实现目标

基于 React 或 Vue 前端框架，结合 Express 后端框架与 MySQL 数据库，构建一套支持服务端渲染（SSR） 的博客系统。核心目标是通过 SSR 技术提升博客文章列表页与详情页的首屏加载速度、SEO 友好度；同时搭建完整的后端服务，提供文章增删改查（CRUD）API 以支撑业务流转。
在此基础上，鼓励探索 HTTP 缓存、Redis 优化、服务端降级、AI 写作助手等进阶能力，打造高性能、高可用、高体验的博客系统。

## 功能列表

### 基础功能

- SSR 核心功能：
  - 实现文章列表页 SSR：服务端预渲染文章列表数据（标题、摘要、发布时间、作者等），返回完整 HTML 给客户端，确保首屏快速呈现与搜索引擎抓取。
  - 实现文章详情页 SSR：根据文章 ID 由服务端预渲染完整文章内容（标题、正文、标签、阅读量等），客户端接收后完成 hydration 激活交互。
- 后端 API 功能：
  - 文章查询：提供列表查询（支持分页、排序）、详情查询（按 ID）接口。
  - 文章新增：接收标题、正文、标签等参数，验证后存入 MySQL 数据库。
  - 文章修改：支持按 ID 更新文章内容、标签、状态（发布 / 草稿）。
  - 文章删除：提供逻辑删除（标记删除状态）或物理删除接口，支持批量删除。
- 数据库设计：
  - 设计合理的 MySQL 数据表（如文章表、用户表、标签表），定义字段类型、主键、外键及索引，确保数据存储规范。
- 前端基础交互：
  - 客户端激活后，支持文章列表分页切换、详情页返回列表、标签筛选等基础交互。

### 进阶功能

- HTTP 缓存策略：
  - 静态资源（CSS、JS、图片、字体等）配置强缓存（Cache-Control、Expires），设置合理缓存时长。
  - 动态资源（文章列表、详情页 HTML）配置协商缓存（ETag、Last-Modified），减少重复数据传输。
- Redis 缓存优化：
  - 为文章查询接口（列表、详情）集成 Redis 缓存，缓存热点数据（如高频访问的文章、首页列表）。
  - 实现缓存更新策略（如文章修改 / 删除时主动失效缓存），避免数据不一致。
- 服务端降级方案：
  - 服务端预取数据（MySQL/Redis 查询）失败时（如数据库宕机、网络异常），返回基础 HTML 骨架，由客户端接管数据请求与渲染。
  - 降级时保留核心交互（如 “刷新重试” 按钮），提示用户当前状态，保障基础可用性。
- AI 写作助手：
  - 在文章发布后台集成 AI 写作功能，支持根据用户输入的标题或关键词，生成文章初稿、摘要或段落建议。
  - 支持对生成内容进行编辑、修改后直接提交发布，提升写作效率。

### 自由探索

例如：
- 前端层面：实现暗黑模式切换、文章评论功能、阅读量统计与展示、标签云可视化等。
- 后端层面：添加用户认证（JWT 登录）、权限控制（管理员 / 普通用户）、接口限流防刷、数据备份功能。
- 性能优化：前端代码分割（Code Splitting）、图片懒加载（SSR 兼容）、MySQL 查询语句优化。
- 拓展功能：支持 Markdown 编辑器发布文章、文章导出（PDF/Markdown）、相关文章推荐。

## 评估标准

* **功能完整性：权重 40%**
  * SSR 核心功能（列表 + 详情页）是否实现，预渲染与客户端激活是否正常；
  * 文章 CRUD API 是否完整可用，接口响应正常；
  * 进阶功能（缓存、Redis、降级、AI 助手）实现程度（实现项越多得分越高）；
  * 无核心功能缺失或严重 Bug。
* **技术规范性：权重 30%**
  * SSR 架构设计合理（如前端打包配置、服务端路由与客户端路由一致性）；
  * 代码规范（命名统一、注释清晰、结构分层合理、无冗余代码）；
  * 数据库设计规范（字段归整、索引优化、关联逻辑正确）；
  * 缓存策略、降级方案逻辑严谨，无数据一致性或可用性问题；
  * 依赖管理规范（无冗余依赖、版本兼容）。
* **用户体验：权重 10%**
  * 首屏加载速度快（SSR 优势体现，无长时间白屏）；
  * 交互流畅（客户端渲染无卡顿，分页、筛选等操作响应及时）；
  * 降级状态下用户体验友好（提示清晰、可操作）；
  * AI 写作/助手类内容直观与交互便捷性。
* **文档表达：权重 10%**
  * 提供完整的项目说明文档（含环境搭建、启动步骤、接口文档、目录结构说明）；
  * 技术选型说明（如 React/Vue 选择理由、SSR 实现方式、缓存策略设计思路）；
  * 文档结构清晰、语言通顺、无歧义，便于查看与引用。
* **创新与拓展：权重 10%**
  * 自由探索功能的实用性与创新性（如搜索交互设计、性能优化手段）；
  * 对 SSR 技术的深度应用（如个性化内容渲染、SSR 与 CSR 混合渲染策略）。
